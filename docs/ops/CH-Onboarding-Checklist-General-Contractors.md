# Onboarding Checklist — General Contractors (v1)

Purpose: Standard onboarding flow for general contractor/renovation members  
Target onboarding window: 10–21 days

---

## 1) Pre-Kickoff (Day 0)
- [ ] Signed membership agreement received
- [ ] Billing setup completed
- [ ] Primary owner and operations contact confirmed
- [ ] Support and escalation channel confirmed

## 2) Kickoff Call (Day 1)
- [ ] Map quote -> deposit -> progress billing -> final invoice flow
- [ ] Confirm top 5 active projects and status
- [ ] Confirm top admin bottlenecks (collections, paperwork, handoffs)
- [ ] Define 30-day success outcomes
- [ ] Confirm scope and partner routing boundaries

## 3) Data and Access Collection (Days 1–5)
- [ ] Open AR/invoice aging imported
- [ ] Active quote/estimate pipeline imported
- [ ] Standard contract/template set collected
- [ ] Change-order documentation process captured
- [ ] Accounting system access granted (or exports provided)
- [ ] Key vendor/subcontractor list captured

## 4) Workflow Setup (Days 4–10)
- [ ] Estimate/invoice templates normalized
- [ ] Deposit/progress/final billing checkpoints configured
- [ ] AR follow-up cadence configured
- [ ] Change-order tracking checklist configured
- [ ] Weekly project admin review rhythm scheduled
- [ ] SLA routing and escalation triggers activated

## 5) Partner Coordination (Days 7–14)
- [ ] Accounting partner handoff process tested
- [ ] Legal intake and contract-routing process tested
- [ ] Escalation ownership matrix documented
- [ ] Compliance/recordkeeping boundaries confirmed

## 6) Quick-Win Validation (Days 10–21)
- [ ] First AR cleanup cycle completed
- [ ] First progress billing run reviewed
- [ ] Highest-risk overdue accounts actioned
- [ ] Member receives first operations summary and next actions

## 7) 30-Day Success Review
- [ ] AR aging improvement baseline captured
- [ ] Invoice cycle-time baseline captured
- [ ] Open legal/accounting items status reviewed
- [ ] Next-30-day project-admin optimization plan agreed

---

## Segment-Specific Risks to Watch
- Irregular project cashflows increase support complexity
- Missing change-order records can cause disputes and delayed billing
- Owner availability can be limited during active job cycles

## Segment-Specific KPI Focus
- AR aging reduction
- Progress invoice cycle-time
- % projects with complete billing documentation
- Collections follow-up completion rate
